Modelo 3: RUP

RUP es una metodología que tiene como objetivo ordenar y estructurar el desarrollo de software, con fases iterativas diseñadas para la creación de proyectos complejos.

RUP tiene diferentes principios fundamentales: Desarrollo Iterativo, Gestión de Requisitos, Arquitectura basada en componentes y Pruebas continuas.

El Desarrollo Iterativo permite hacer ajustes al avanzar con el proyecto.

La Gestión de requisitos define todo lo que se va a necesitar desde un inicio.

La Arquitectura es la estructura para facilitar la creación del producto.

Las Pruebas aseguran una buena calidad para el cliente.

Características:

1. Forma disciplinada de asignar tareas y responsabilidades (quién hace qué, cuándo y cómo).

2. Pretende implementar las mejores prácticas en Ingeniería de Software.

3. Desarrollo iterativo.

4. Administración de requisitos.

5. Uso de arquitectura basada en componentes.

6. Control de cambios.

7. Modelado visual del software.

Roles en RUP: Los roles principales y específicos de RUP son iguales que Prototipos:

Analista: recopila toda la información inicial requerida por el cliente.

Diseñador: desarrolla la estructura del proyecto.

Codificador: implementa toda la información recopilada.

Tester y Gestores del proyecto: prueban la funcionalidad del programa y supervisan el avance y cumplimiento de plazos.

Fases:

1. Fase de Inicio.

2. Fase de Elaboración.

3. Fase de Construcción.

4. Fase de Transición.

Beneficios de RUP:

Reduce los riesgos del proyecto desde etapas tempranas.

Garantiza buena calidad gracias a constantes revisiones y pruebas.

Es adaptable según el proyecto.

Facilita la documentación de resultados.

Ventajas:

1. Forma disciplinada de asignar tareas y responsabilidades.

2. Mantenimiento más sencillo.

3. El conocimiento adquirido en una iteración puede aplicarse a las siguientes iteraciones.

4. Los usuarios están involucrados continuamente.

Desventajas:

1. Por el grado de complejidad puede no resultar muy adecuado.

2. El RUP es generalmente mal aplicado en el estilo cascada.

3. Requiere conocimientos del proceso.

4. Genera muchos costos.
