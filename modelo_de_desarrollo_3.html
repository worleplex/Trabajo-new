Modelo RUP
RUP es una metodología que tiene como objetivo ordenar y estructurar el desarrollo de software. El cual contiene fases iterativas y son diseñadas para la creación de proyectos complejos.
RUP tienen diferentes principios fundamentales y estos son: Desarrollo Iterativo, Gestión de Requisitos, Arquitectura basada en componentes y Pruebas continuas. El Desarrollo iterativo permite hacer ajustes al avanzar con el proyecto, la Gestión de requisitos define todo lo que se va a necesitar desde un inicio, la Arquitectura es la estructura para facilitar la creación del producto y las Pruebas nos aseguran una buena calidad para el cliente.
Los roles principales y específicos de RUP son iguales que Prototipos: Analista quien recopila toda la información inicial requerida por cliente, Diseñador quien va creando la estructura del proyecto, el Codificador es quien implementa toda información recopilada, los Tester y Gestores del proyecto son quienes se encargan de probar la buena funcionalidad del programa y supervisan el avance y cumplimiento de plazos.
Las fases utilizadas en esta metodología son las siguientes: Fase de Inicio, Fase de Elaboración, Fase de Construcción y Fase de Transición.
Beneficios de RUP para estudiantes de doctorado:
Reduce los riesgos del proyecto desde etapas tempranas.
Tiene una buena calidad por las constantes revisiones y pruebas.
Es adaptable según el proyecto.
Facilita la documentación de resultados.
